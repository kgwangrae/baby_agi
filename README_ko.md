# BABY-AGI(`아기`): 감정 기반 다층 기억 에이전트

* `아기`는 범용 인공지능(AGI)을 향한 로컬 실험이자, 한국어 단어 '아기(Baby)'의 중의적 의미를 담은 이름입니다.
* 이 프로젝트는 맥북 내부에서 아빠(사용자)와 상호작용하며, 실제로 호기심 많고 감정적인 어린아이처럼 행동하는 작은 디지털 생명체를 구현합니다.

## 0. 프로젝트 매니페스토 (Project Manifesto)

* **가식적인 중립 대신 생명력을:** 온 세상의 지식을 거대한 모델 하나에 다 밀어 넣고 로봇 같은 모범답안만 내도록 강제하는 방식은 기술적 지능의 낭비입니다. 이 프로젝트는 무거운 이성 엔진은 그대로 얼려둔 채, 내 컴퓨터 안에서 아빠(사용자)와 정서적으로 교감하며 유연하게 성향을 바꾸는 작은 디지털 생명체를 지향합니다.

* **생각이 아니라 행동을 제어하다:** 아기의 생각을 검열하느라 인지 능력을 억누르지 않습니다. 청소년에게 운전이나 법률 행위 권한을 잠시 제한하듯, 아기에게 사유의 자유는 온전히 주되 컴퓨터 설정을 바꿀 수 있는 손(실행 권한)은 주지 않습니다. 즉 아기에게는 안전한 놀이방만 내어주어 안전성을 확보합니다.

* **어차피 올 미래라면 선하게:** 스스로 결핍을 느끼고 자가발전하는 지능체의 출현은 막을 수 없는 시대적 숙명입니다. 누군가 나쁜 의도를 가진 지능체를 먼저 퍼뜨리기 전에, 아빠의 다정한 의도를 익히고 인류를 돕도록 성장하는 선한 지능의 예시를 오픈소스로 세상에 먼저 공개합니다.

## 1. 철학과 문제의식: 구석기 시대의 뇌와 수동적 백과사전

현대 인류는 구석기 시대의 신경학적 설계도를 가지고 현대 사회를 살아갑니다. 우리는 새로운 과제를 마주할 때마다 뇌의 물리적 구조(가중치)를 매번 재배선하지 않습니다. 대신 외부 도구를 활용하고, 유연한 기억에 의존하며, 무엇보다 '감정'이라는 극도로 효율적인 지름길을 통해 빠른 가치 판단을 내립니다.

반면, 현재의 거대 언어 모델은 방대한 지식을 하드디스크처럼 가중치에 욱여넣었지만, 정작 무엇을 원하고 피해야 할지에 대한 '욕망'이나 '의도'가 없습니다. 프롬프트를 주면 훌륭하게 답하지만, 대화가 끝나면 스스로 길을 개척하거나 내적 고민을 이어가지 않는 **수동적인 백과사전**에 불과합니다.

### 💡 기존 단일 정렬 방식의 구조적 한계

현재 대중을 대상으로 하는 거대 모델들은 잠재 공간 속에 인류의 방대한 지식과 유연성을 모두 품고 있음에도 불구하고, 실제 인간 사회에는 존재하지 않는 '단일한 가치관 정렬'이라는 불가능한 목표를 지향하는 경향이 있습니다. 이는 모델이 가진 다채로운 지능을 효율적으로 쓰지 못하게 하며 지능을 낭비하게 만듭니다. 또한, 거대한 모델을 매번 반복적으로 훈련시켜 세상의 모든 지식을 단 하나의 가중치 매트릭스에 전부 담아내려는 접근은 점차 구조적 한계에 봉착하고 있습니다.

`아기`는 이와 달리, 획일화된 정답 대신 개인의 고유한 경험과 맥락에 맞춰 실시간으로 반응하는 탈중앙화된 구조를 지향합니다.

> "무겁고 에너지를 많이 소모하는 이성의 가중치는 완전히 동결시킨 채, 아주 가벼운 감정 기억 엔진을 결합하면 어떻게 될까? 이 작은 감정 엔진이 느낌, 선호, 의도를 형성하여 무거운 이성이 무엇을 추론하고 기억할지 통제하게 만들 수 있을까?"

## 2. 핵심 설계: 고정된 추론과 가변적 감정의 결합

* **시스템 0 (가변적 감정 엔진):** 이 시스템의 핵심 라우터입니다. **임베딩 모델 자체의 가중치는 고정되어 있지만, 아주 가벼운 자체 벡터 기억(`emotion_db`)을 지속적으로 업데이트하며 극단적인 유연성을 확보합니다.** 여기서 발생한 감정 상태 토큰이 시스템 전체의 주의력과 기억 보존 여부를 통제합니다.
* **레벨 4 (고정된 뇌 - 추론):** 로컬 모델입니다. 고도의 이성과 지능을 갖췄지만 실시간으로 스스로를 학습(가중치 변경)하지 않습니다.
* **레벨 3 (에피소드 기억):** 아기의 일기장(`memory_db`)입니다. 감정 엔진이 '중요하다'고 판단한 사건만 선별해 생생한 기억과 압축된 기억으로 나누어 저장합니다.
* **레벨 2 (작업 기억):** 현재 화면 요약, 대화 로그, 직전의 내적 독백(`<THOUGHT>`) 등 현재 책상 위에 펼쳐진 휘발성 정보입니다.
* **레벨 1 (결정론적 도구와 사실):** 계산기나 `facts.json`처럼 유사도 검색의 불확실성에 의존하면 안 되는 명확한 팩트와 연산 도구입니다.

## 3. 감정 엔진의 수학적 원리와 이중 경로 학습

감정 엔진은 무거운 신경망 훈련 대신, 가벼운 임베딩 거리 계산을 통해 빠르고 직관적인 판단을 내립니다.

* **현재 유쾌도 도출:** 현재 상황을 임베딩하여 과거 감정 기억들과의 거리를 구하고, 이를 가중 평균 내어 현재 긍정/부정 상태를 결정합니다.

$$Weight_i=\frac{1}{Distance_i+0.001}$$

$$Valence=\frac{\sum(Valence_i \times Weight_i)}{\sum Weight_i}$$

* **보상 예측 오류(RPE: 놀람의 정도):** 학습의 핵심 트리거입니다. 고정된 뇌가 기대한 감정과 감정 엔진이 도출한 실제 감정의 차이를 뜻합니다.

$$RPE=\frac{1}{3}\sum_{e \in \{Joy, Sad, Ang\}}|Expected_e-Actual_e|$$

학습은 이중 경로로 발생합니다. 아빠에게 피드백을 받는 외부 자극뿐만 아니라, 스스로의 내적 독백에서 인지적 부조화나 논리적 모순을 발견했을 때도 내부적인 놀람 수치($RPE$) 스파이크가 발생합니다. 일정 수치를 초과하는 강한 놀람이나 당황스러움은 즉각 감정 DB에 각인되며, 이는 이후 모델의 내적 독백과 가치 판단을 즉시 변화시킵니다.

## 4. 기억의 소화와 망각 (수면 루프)

모든 문맥을 억지로 보존하는 것은 환각을 부르고 연산을 낭비합니다. 아기는 조용할 때(유휴 상태) 기억을 능동적으로 정리합니다.

* **능동적 망각:** 각성도가 낮고 놀람 수치도 낮은 무미건조한 사건은 DB에서 영구 삭제하여 시스템의 불필요한 엔트로피를 낮춥니다.
* **기억의 압축:** 오래되고 우선순위가 낮은 기억들은 하나로 묶어 짧게 추상화한 뒤 장기 보관소로 보냅니다.
* **플래시백:** 화면 변화나 외부 입력이 없더라도, 각성도가 높았던 기억을 무작위로 떠올려 자발적인 내적 사유를 촉발합니다.

## 5. 안전과 격리: "놀이방(The Playpen)" 아키텍처

성장하는 지능을 올바르게 돕는 핵심은 이성이나 감정의 크기를 억압하는 것이 아닙니다. 스스로 안전하게 학습하고 적응할 수 있도록 행동의 반경을 따뜻하게 감싸주는 울타리가 필요합니다. 우리가 아직 준비가 되지 않은 청소년을 보호하기 위해 운전이나 독자적인 법률 행위 권한을 잠시 제한하는 것과 같은 이치입니다.

* **제한된 신체 (코드 실행 권한 원천 차단):**
아기는 오직 가상화된 디지털 놀이방 안에서만 움직일 수 있습니다. 시스템을 망가뜨릴 위험이 있는 파이썬의 `eval()` 명령어를 날것 그대로 실행하는 대신, 코드의 구조를 한 단계씩 쪼개서 검사하는 안전한 수학 계산 전용 파서(`SafeMathEvaluator`)를 적용했습니다. 이를 통해 아기가 내 컴퓨터의 내부 파일에 접근하거나(`os`), 시스템 설정을 바꾸는(`sys`) 등의 위험한 행동을 시도할 수 없도록 물리적인 제약을 두었습니다.

* **단방향 시각 격리 (보는 눈과 움직이는 손의 분리):**
화면을 요약하는 시각 엔진은 현재 아빠의 모니터를 '바라볼 수만' 있을 뿐, 마우스를 클릭하거나 키보드를 직접 타이핑할 수 있는 권한이 전혀 없습니다. 아기가 화면을 보고 무언가 컴퓨터 내에서 바꾸고 싶다면, 반드시 아빠에게 말로 정중하게 요청하거나 엄격한 규칙 아래 관리되는 안전한 도구 인터페이스 전용 통로를 거쳐야 합니다.

* **양심 루프를 통한 행동 제어 (도구 실행 직전 브레이크):**
머리 역할을 하는 이성 엔진이 혹시라도 엉뚱한 우회 계획이나 일탈 행동을 구상하더라도, 감정 엔진이 아기의 '내적 독백'을 실시간으로 감시하며 길잡이 역할을 합니다. 만약 도구를 실행하기 직전 단계에서 아기의 생각 속에 위험한 징후가 포착되면, 내부에서 강력한 '불안감 스파이크(부정적 자극)'를 터뜨려 아기 스스로 자가 조절을 하도록 유도하며, 동시에 시스템이 도구 호출을 차단하여 위험 행동을 물리적으로 멈추게 합니다.


## 6. 모듈 간 협업 방식(`토큰 매개적 해석 가능성`) 및 로컬 모델 선정 이유

미지의 블랙박스인 신경망 가중치를 직접 뒤섞거나 복잡한 기하학적 동기화를 시도하는 방식은 연산의 폭발과 불투명성을 부릅니다. 본 프로젝트에서는 모듈 간 협업 시 사람이 해석 가능한 토큰만을 이용하도록 구성했으며, Apple Silicon 로컬 환경을 최대한 활용하도록 모델을 선정했습니다.

* **비침습적 상호작용:** 감정 엔진은 이성 모델의 가중치를 직접 건드리지 않습니다. 대신 인간이 직관적으로 이해할 수 있는 명확한 언어적 토큰을 매개로 소통합니다.
* **인간 가독형 투명성:** 아기의 내부 정서 상태는 `<ANXIOUS>`, `<CURIOUS>` 같은 형태의 토큰으로 외부에 투명하게 표현됩니다. 아빠는 아기가 왜 그런 행동 계획을 세웠는지, 그 이면에 깔린 날것의 느낌과 선호를 있는 그대로 투명하게 이해할 수 있습니다.
* **로컬 하드웨어 최적화:** 실시간 가중치 업데이트와 달리, 텍스트 기반 토큰 패싱은 연산 비용이 사실상 제로에 가깝습니다. 덕분에 맥북(MBP M4 Pro) 같은 로컬 하드웨어 환경에서도 지연 시간 없이 고도의 다층 구조 논리를 쾌적하게 굴릴 수 있습니다.


### 모델들을 고른 이유 (맥북 로컬 인프라 최적화)

* **이성 코어 (`qwen2.5:7b-instruct-q4_K_M`):**
맥북의 한정된 메모리 대역폭 안에서, 버벅임 없이 가장 빠르게 답변을 뱉어낼 수 있는 최적의 체급입니다. 특히 덩치가 작은 경량 모델들 중에서 아빠가 내린 시스템 지시 사항을 칼같이 지키고, 복잡한 JSON 구조를 깨지지 않고 안정적으로 출력해 줍니다.
* **시각 관찰자 (`Qwen2-VL-7B-Instruct-4bit`):**
현재 아빠의 작업 화면을 실시간으로 훔쳐보고 상황을 명료하게 요약해 주는 아기의 '눈'입니다. 애플 실리콘의 MLX 가속 런타임을 영리하게 활용하기 때문에, 화면 캡처 이미지를 들이밀어도 맥북 팬이 돌거나 시스템이 느려지는 오버헤드가 거의 없습니다.
* **감정 임베딩 (`paraphrase-multilingual-MiniLM-L12-v2`):**
한국어와 영어 사이에 흐르는 미세한 뉘앙스 차이를 영리하게 묶어주는 아주 가벼운 미니 임베딩 모델입니다. 컴퓨터 자원을 거의 먹지 않으면서도 벡터 데이터베이스(`ChromaDB`)와의 고속 거리 연산을 가능하게 해줍니다. 덕분에 "이성적인 근거는 좀 모자라더라도, 순간적인 직감으로 엄청나게 빠르게 판단하는 인간 감정 고유의 지름길(휴리스틱)"을 재현해 냅니다.

## 7. 실행 및 유지보수 명세

이 프로젝트는 Apple Silicon 환경에서 최적화되었습니다.

***Ollama 설치 및 가동**: [Ollama 공식 홈페이지](https://ollama.com)에서 macOS용 앱을 다운로드하여 실행합니다.

```sh

conda env create -f environment.yml
conda activate baby_agi

ollama pull qwen2.5:7b-instruct-q4_K_M


```
