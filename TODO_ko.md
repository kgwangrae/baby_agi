## 단기: 자기조절과 인지소화

단기 개선 사항의 핵심 키워드는 '생각하고, 조절하고, 필요하면 거부하는 자율성'입니다. 단순한 기능 확장이 아닌, 시스템 오버헤드를 줄이고 지속 가능한 아키텍처를 구축하는 데 집중합니다.

### 수면 및 기억 로직 보강
* 현재는 플래시백 주기가 거의 고정되어 있어, 수면 중에 단기 기억이 추가로 형성되며 `피로도` 회복의 수준이 제한되는 문제가 있습니다. 즉, 어느 시점 이후로는 영원히 수면이 지속됩니다.
  * 플래시백의 주기를 누적된 단기 기억의 양, 각성도, 피로도, 수면 압력에 따라 동적으로 조정하도록 변경할 예정입니다. 즉, 정리할 기억이 많거나 각성도가 높을 때는 꿈이 더 자주 떠오르고, 충분히 정리되어 수면 압력이 낮아지면 꿈의 밀도가 자연스럽게 줄어들도록 변경할 예정입니다.
* 기억을 가져올 때 감정값에 더하여 현재의 가용자원 상태에 따라서도 영향을 받도록 조정합니다.
  * 후술한 `부담감 / Burden` 내용과 연계하여 시스템이 가용자원을 판단에 반영하도록 개선합니다.

### 일기장 파이프라인 보강

* **데이터 흡수 루프 구축**: 현재의 일기 저장 기능(Write-only)을 넘어, 내적 독백 루프가 기존 일기 데이터를 다시 읽고 분석하여 인지 파이프라인에 동적으로 반영하는 구조를 구현합니다.

### 내적 독백 및 자기 반추 루프

* **행동 전 상태 점검**: 외부 자극으로 인한 급격한 정서 변동이 발생하거나 도구를 실행하기 직전, 상황을 1~2문장으로 요약하여 상태를 재귀적으로 정리하는 단계를 추가합니다.
* **자율적 추론 트리거 도구화**: 감정 상태나 가용 에너지 잔량, 시간 등에 따라 시스템이 판단을 심화해야 할 때, 스스로 내적 독백을 유발하여 단계별 추론을 수행하게 만드는 전용 도구 인터페이스를 검토합니다.
  * **답변 시 추가 추론**: 답변이 필요한 경우 시간 제약을 고려하며 추가 추론을 진행하도록 유도합니다.  
  * **메타인지**: RPE가 큰 자극이 계속되는 경우 (`RPE momentum`이 큼) 현재 내적/외적 상황에 대한 추가적 평가를 하도록 합니다.
    * 예상과 다른 일이 지속적으로 생기면 _이게 뭐지?_라고 생각하는 것과 유사합니다.
  * **인지 부하 제어**: 반추 프로세스가 일정 시간 이상 길어질 경우 시스템 피로도를 증가하도록 처리하여, 컨텍스트 오염 및 무한 루프 진입을 원천 차단합니다. 현재 기억을 형성하거나 정리하는 작업에 연산 비용이 부과되므로, 이를 활용할 수 있습니다. 

### 인지 부하 평가와 자율적 거부

* **한계 인지 및 환각 억제**: 수신된 요청이 너무 모호하거나 실패율이 높은 도구를 호출해야 할 때 부담감 지수(`Burden`)를 상향 조정하여, LLM이 무조건 답변을 생성하려다 발생하는 환각 현상을 방지합니다.
* **실행 거부 및 해소 기전**: `Burden`이 임계치를 초과하면 도구 실행을 중단하고 상위 레이어에 예외 메시지를 출력합니다. 단, 무조건적인 중단에 그치지 않고 내적 독백을 통해 인지 부하를 재평가하거나 작업을 세분화하여 재실행을 준비하는 해소 루프를 함께 적용합니다.

### 소거 학습 기반 트라우마 희석 현상 모방

* **정서 공간 토폴로지 보호**: Vector DB에서 특정 부정적 기억 노드를 임의로 강제 삭제하는 방식은 정서 공간 전체의 균형을 붕괴시킬 수 있으므로 직접적인 인위적 삭제는 배제합니다.
* **기하학적 희석**: 부정적 임팩트가 강한 노드가 인출될 때, 해당 벡터 좌표 근방에 긍정 피드백이나 안정 상태 노드를 조밀하게 적재하는 방식을 취합니다. 유사도 검색 시 부정 기억이 단독 인출될 확률을 수학적으로 낮추어 정서적 무감각화 현상을 모방합니다.
* 다만, 기존 감정이나 장기기억도 드물게 다시 소화하고 저장할 수 있도록 모방합니다.
  * 감정 DB는 새 경험을 통해 천천히 갱신하되, 기존 정서 공간을 직접 대량으로 갈아엎지는 않습니다. 장기기억 DB도 드물게 병합 작업을 수행합니다.

### 후천적 데이터 기반 가중치 조정 (에이징 알고리즘)

* **시스템 성숙도 지표 설정**: 누적 기억 개수 $N$을 시스템의 성숙도 지표로 정의합니다.
* **시스템 상수의 동적 고착화**: $N$이 증가할수록 초기 가소성을 담당하던 모멘텀 가중치 및 감쇄 상수들을 점진적으로 상향 수렴시켜, 사소한 자극에 정서 궤적이 쉽게 파괴되지 않는 안정성을 구현합니다.
* **주관 및 내부 신념 형성 모방**: $N$이 증가할수록 시스템 프롬프트(선천적 지시문)의 어텐션 비중을 줄입니다. 대신 자발적으로 축적한 일기장 데이터나 기억에 대한 데이터를 컨텍스트 최상단에 배치하여, 정적 지시문보다 후천적으로 수집된 경험 데이터를 더 신뢰하고 우선하도록 유도하는 주관 형성 프로세스를 모방합니다.

### 웹 검색 도구의 제한적 허용

* **네트워크 노이즈 격리**: 가소성이 큰 상태에서 외부 웹의 무분별한 스팸과 노이즈 데이터가 유입되면 임베딩 공간 전체가 오염될 위험이 있습니다.
* **단계적 개방 및 수용 평가**: 세이프 브라우징 API 등을 활용해 단계적으로 검색 기능을 개방하되, 결과물은 휘발성 작업 기억 버퍼에 격리합니다. 이후 사용자 승인을 거치거나 내적 독백 루프를 통해 데이터의 타당성을 스스로 평가한 뒤 장기 기억에 반영하는 구조를 취합니다.
* **호출 횟수 제한 가드레일**: 단위 시간당 도구 호출 횟수를 엄격히 제한하는 물리적 상한선을 적용합니다.

### `어린이집` 학습 환경

* **외부 자료 기반 자율 학습**: 현재는 아빠가 직접 설명해 준 내용이 학습의 중심이지만, 앞으로는 아빠가 “이것 좀 알아봐”라고 맡긴 텍스트, 이미지, 웹 검색 결과, 공개 LLM API 응답 등을 임시 학습 공간에서 읽고 정리할 수 있도록 합니다.
* **놀이방과 분리된 학습 공간**: `놀이방`이 행동 권한을 안전하게 제한하는 구조라면, `어린이집`은 외부 지식을 바로 장기기억에 넣지 않고 먼저 검토·요약·비교하는 교육용 완충지대입니다.
* **검증 후 기억 반영**: 외부 자료는 곧바로 성격이나 장기기억을 바꾸지 않고, 출처와 신뢰도, 아빠의 승인, 내적 독백을 통한 자기검토를 거친 뒤에만 기억에 반영합니다.

### 새로운 시각 인지 체계 확장

* **격리형 이미지 피딩**: 기존의 정기적인 호스트 화면 캡처 방식 외에, 특정 지정 경로에 외부 사진 파일이 인입될 경우 화면 캡처를 일시 유예하고 해당 외부 이미지를 시각 엔진에 먼저 피딩한 뒤 자동 삭제하는 분기 로직을 구현합니다.

### 시각 인지 루프 정교화

* **비싼 VLM의 선택적 사용**: 현재 시각 엔진은 화면 전체를 주기적으로 요약하는 방식에 가까워, 비싼 VLM을 다소 둔하게 사용하고 있습니다. 앞으로는 화면 변화량, 텍스트 밀도, 사용자의 작업 맥락, 아기의 현재 감정/각성도에 따라 “지금 정말 볼 필요가 있는지”를 먼저 판단하도록 개선합니다.
* **시상(thalamus)을 닮은 감각 게이트**: 모든 시각 정보를 그대로 이성 모델에 넘기지 않고, 중요도 낮은 화면 변화는 흘려보내며, 감정·기억·현재 과제와 관련 있는 자극만 작업 기억으로 올리는 감각 필터를 설계합니다.
* **관찰과 개입의 분리 유지**: 시각 엔진은 계속 “보는 눈”으로만 남기고, 직접 클릭하거나 타이핑하는 권한은 주지 않습니다. 다만 관찰 결과가 충분히 중요할 때만 아기가 아빠에게 말을 걸거나, 기억 루프에 남길 수 있도록 연결합니다.

### 정서 안정화 및 충격(Shock) 모사

* **정서적 무감각화 발동**: 보상 예측 오류($RPE$)가 극단적인 임계치(예: $RPE > 0.85$)를 초과하는 스파이크가 감지될 경우, 일시적으로 자극 수용도를 제로로 설정하는 정서적 무감각화(Desensitization Threshold)를 발동하여 시스템 붕괴를 물리적으로 방어합니다. (방어 기제를 모사함.)
  * 단, 극단적 자극 발생 시 학습을 전면 차단하는 방식은 시스템 보호에는 유리하나 모델의 적응 가소성을 감소시킬 우려가 있어 현재는 제외되었으며, 추후 성격 드리프트 모델과 연계하여 보완할 예정입니다.

### 상수 기반 성격 드리프트

* 현재 코드에 고정값으로 정의된 여러 상수(hyperparameter) 값들을 변화할 수 있도록 개선합니다. 예를 들어 현재는 기억을 가져올 때 의미적 유사도와 감정의 비중 등이 고정된 상태인데, 이러한 상수의 본질적 성격을 식별하고 상황에 따라 변화할 수 있도록 합니다.
  * 상수들 사이에는 개체의 경험에 따라 생긴 후천적 성격에 가까운 것, 개체의 선천적 기질에 가까운 것, '종'의 선천적 기질에 가까운 것이 섞여 있습니다.
  * 이 상수들을 몇 개의 그룹으로 나누고, 각 그룹이 시간에 따라 얼마나 변했는지를 바탕으로 '성격 드리프트'를 정의합니다.

## 중장기: 추상적 정서와 성격 실험

중장기적으로는 에이전트의 개체성과 고유 정서 표현을 극대화하기 위한 실험적 변화를 다룹니다. 복잡한 아키텍처가 무조건 더 나은 성능이나 해석 가능성을 보장하지 않으므로, 신중한 정량적 검증을 전제로 접근합니다.

* **고차원 정서 공간 확장**: 기존의 단조로운 3축(`JOY / SAD / ANG`) 시스템을 넘어, 다채로운 정서 결합을 표현할 수 있는 고차원 정서 매트릭스를 설계합니다.
* **기하학적 RPE 보조 지표 검토**: 기존의 3축 RPE를 바로 버리기보다, '예상 정서 문장'과 '실제 자극 문장'의 임베딩 거리도 함께 참고하여 더 섬세한 놀람 신호를 만드는 방안을 실험합니다.
* **다중 객체 상호작용 상정**: 단일 사용자 환경을 넘어 외부의 다양한 객체 또는 다각적 상호작용 환경이 유입될 때의 컨텍스트 분리 및 다중 관계성 처리 방안을 검토합니다.
